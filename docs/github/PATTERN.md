### Commits (em inglês)
Formato obrigatório:

```
fix: short and objective change
feat: short and objective change
refactor: short and objective change
docs: short and objective change
chore: short and objective change
```

Exemplos:
- `fix: handle empty unit title`
- `feat: generate lesson plan via AI`
- `refactor: extract rag service`

### Pull Request (em inglês)
Template obrigatório:

**Description**  
One clear sentence about what was delivered.

**Type of change**  
- [ ] Configuration/Build  
- [ ] Bug fix  
- [ ] New feature  
- [ ] Refactor  
- [ ] Documentation  

**Main changes**  
- bullet points  

**How to test**  
1. steps  

**Checklist**  
- [ ] items  

### Merge (em inglês)
Formato obrigatório:

```
merge: <PR title>
```

Opcional com motivo curto:

```
merge: <PR title> (short reason)
```

### Branchs (em inglês)
Formato obrigatório:

```
<type>
```

Exemplos:
- `feat`
- `fix`
- `refactor`
- `docs`
- `chore`
- `test`
